# وثيقة المتطلبات - Student Projects Hub

## مقدمة

Student Projects Hub هو موقع إلكتروني يتيح لطلاب الجامعات نشر مشاريعهم الأكاديمية والبرمجية ومشاركتها مع المجتمع. يعتمد الموقع على HTML وCSS وJavaScript في الواجهة الأمامية، وSupabase كقاعدة بيانات وخدمة مصادقة. يمكن لأي زائر تصفح المشاريع والتفاعل معها، بينما تقتصر إمكانية النشر على الطلاب المسجلين بإيميل جامعي.

---

## قاموس المصطلحات

- **النظام (The System)**: تطبيق Student Projects Hub الكامل (الواجهة + Supabase)
- **الزائر (Visitor)**: أي شخص يتصفح الموقع دون تسجيل دخول
- **الطالب (Student)**: مستخدم مسجل بإيميل جامعي موثق
- **الأدمن (Admin)**: مستخدم ذو صلاحيات إدارية كاملة
- **المشروع (Project)**: محتوى ينشره الطالب يشمل العنوان والوصف والروابط والصور
- **الإعجاب (Like)**: تفاعل من زائر أو طالب على مشروع، مقيد بجهاز واحد
- **البصمة الرقمية (Device Fingerprint)**: معرف فريد مشتق من خصائص الجهاز والمتصفح
- **الإيميل الجامعي (University Email)**: إيميل ينتهي بنطاق أكاديمي معروف (مثل .edu أو .ac.*)
- **الحظر (Ban)**: منع حساب طالب من تسجيل الدخول وإجراء أي عمليات

---

## المتطلبات

### المتطلب 1: عرض المشاريع للزوار

**قصة المستخدم:** بصفتي زائراً، أريد تصفح مشاريع الطلاب من الصفحة الرئيسية، حتى أتمكن من الاطلاع على أعمالهم دون الحاجة إلى تسجيل دخول.

#### معايير القبول

1. THE System SHALL عرض قائمة المشاريع المنشورة في الصفحة الرئيسية لجميع الزوار
2. WHEN يتم تحميل الصفحة الرئيسية، THE System SHALL جلب المشاريع من قاعدة البيانات وعرضها بشكل بطاقات
3. THE System SHALL عرض في كل بطاقة مشروع: العنوان، الوصف المختصر، اسم الطالب، عدد الإعجابات، وصورة مصغرة إن وُجدت
4. WHEN ينقر الزائر على بطاقة مشروع، THE System SHALL فتح صفحة تفاصيل المشروع
5. THE System SHALL عرض زر تسجيل الدخول في شريط التنقل للزوار غير المسجلين

---

### المتطلب 2: تفاصيل المشروع

**قصة المستخدم:** بصفتي زائراً أو طالباً، أريد رؤية التفاصيل الكاملة لأي مشروع، حتى أفهم محتواه وأتواصل مع صاحبه.

#### معايير القبول

1. WHEN ينقر المستخدم على مشروع، THE System SHALL عرض صفحة تفاصيل تشمل: العنوان، الوصف الكامل، اسم الطالب، رابط GitHub أو رابط المشروع، الصور، وعدد الإعجابات
2. THE System SHALL عرض زر الإعجاب في صفحة تفاصيل المشروع
3. WHEN يُعرض المشروع، THE System SHALL تحميل عدد الإعجابات الحالي من قاعدة البيانات

---

### المتطلب 3: نظام المصادقة بالإيميل الجامعي

**قصة المستخدم:** بصفتي طالباً، أريد تسجيل الدخول عبر GitHub أو Google باستخدام إيميلي الجامعي، حتى أتمكن من نشر مشاريعي بشكل موثوق.

#### معايير القبول

1. THE System SHALL توفير خيار تسجيل الدخول عبر GitHub وGoogle OAuth
2. WHEN يكمل المستخدم المصادقة OAuth، THE System SHALL التحقق من أن الإيميل المرتبط ينتهي بنطاق جامعي معترف به
3. IF كان الإيميل لا ينتهي بنطاق جامعي معترف به، THEN THE System SHALL رفض التسجيل وعرض رسالة خطأ واضحة
4. IF كان الحساب محظوراً، THEN THE System SHALL رفض تسجيل الدخول وعرض رسالة توضح أن الحساب موقوف
5. WHEN يسجل طالب الدخول لأول مرة، THE System SHALL إنشاء سجل مستخدم في قاعدة البيانات
6. WHEN يسجل الطالب الدخول بنجاح، THE System SHALL تحديث شريط التنقل لإظهار اسم المستخدم وزر تسجيل الخروج

---

### المتطلب 4: نشر المشاريع

**قصة المستخدم:** بصفتي طالباً مسجلاً، أريد نشر مشروعي عبر واجهة مخصصة، حتى يتمكن الآخرون من الاطلاع على عملي.

#### معايير القبول

1. WHILE الطالب مسجل الدخول، THE System SHALL إظهار زر "نشر مشروع" في شريط التنقل
2. WHEN ينقر الطالب على "نشر مشروع"، THE System SHALL فتح نموذج إدخال بيانات المشروع
3. THE System SHALL قبول الحقول التالية في نموذج نشر المشروع: العنوان (إلزامي)، الوصف (إلزامي)، رابط GitHub، رابط تجريبي، صورة المشروع، والتقنيات المستخدمة
4. WHEN يرسل الطالب النموذج بحقول إلزامية مكتملة، THE System SHALL حفظ المشروع في قاعدة البيانات ونسبه للطالب
5. IF حاول الطالب نشر مشروع بحقول إلزامية فارغة، THEN THE System SHALL منع الإرسال وعرض رسائل خطأ للحقول المطلوبة
6. WHEN يُنشر المشروع بنجاح، THE System SHALL توجيه الطالب إلى صفحة تفاصيل المشروع الجديد

---

### المتطلب 5: تعديل وحذف المشاريع (للطالب)

**قصة المستخدم:** بصفتي طالباً، أريد تعديل أو حذف مشاريعي الخاصة، حتى أتمكن من إبقائها محدثة أو إزالتها عند الحاجة.

#### معايير القبول

1. WHILE الطالب مسجل الدخول وينظر إلى مشروع خاص به، THE System SHALL عرض أزرار التعديل والحذف
2. WHEN ينقر الطالب على "تعديل"، THE System SHALL فتح نموذج مسبق التعبئة ببيانات المشروع الحالية
3. WHEN يحفظ الطالب التعديلات، THE System SHALL تحديث سجل المشروع في قاعدة البيانات
4. WHEN ينقر الطالب على "حذف"، THE System SHALL عرض رسالة تأكيد قبل الحذف
5. WHEN يؤكد الطالب الحذف، THE System SHALL حذف المشروع من قاعدة البيانات وإعادة توجيهه للصفحة الرئيسية
6. IF حاول الطالب تعديل أو حذف مشروع لا يملكه، THEN THE System SHALL رفض العملية وإعادة توجيهه

---

### المتطلب 6: نظام الإعجابات بالبصمة الرقمية

**قصة المستخدم:** بصفتي زائراً أو طالباً، أريد الإعجاب بالمشاريع، مع ضمان أن كل جهاز لا يمنح إعجاباً واحداً فقط لكل مشروع.

#### معايير القبول

1. THE System SHALL توليد بصمة رقمية فريدة لكل جهاز مستناداً على خصائص المتصفح والجهاز
2. WHEN يضغط المستخدم على زر الإعجاب، THE System SHALL حساب بصمة الجهاز وإرسالها مع طلب الإعجاب
3. IF كانت بصمة الجهاز قد أعجبت بهذا المشروع من قبل، THEN THE System SHALL رفض الإعجاب وعرض حالة "تم الإعجاب مسبقاً"
4. WHEN يُسجَّل إعجاب جديد بنجاح، THE System SHALL تحديث عداد الإعجابات فورياً في الواجهة
5. THE System SHALL تخزين الإعجابات في قاعدة البيانات مرتبطةً ببصمة الجهاز ومعرف المشروع
6. WHERE الجهاز قد أعجب بالمشروع، THE System SHALL عرض زر الإعجاب في حالة نشطة (مضيئة)

---

### المتطلب 7: لوحة تحكم الأدمن

**قصة المستخدم:** بصفتي أدمناً، أريد إدارة المشاريع والحسابات، حتى أحافظ على جودة المحتوى وسلامة المنصة.

#### معايير القبول

1. WHILE المستخدم له صلاحيات أدمن، THE System SHALL عرض رابط "لوحة التحكم" في شريط التنقل
2. THE System SHALL توفير صفحة لوحة تحكم تعرض: قائمة جميع المشاريع، وقائمة جميع المستخدمين
3. WHEN ينقر الأدمن على "تعديل" أي مشروع، THE System SHALL فتح نموذج تعديل ذلك المشروع
4. WHEN يؤكد الأدمن حذف مشروع، THE System SHALL حذفه من قاعدة البيانات بغض النظر عن مالكه
5. WHEN يضغط الأدمن على "حذف حساب"، THE System SHALL حذف الحساب وجميع مشاريعه من قاعدة البيانات
6. WHEN يضغط الأدمن على "حظر حساب"، THE System SHALL تعيين حالة الحساب إلى محظور ومنعه من تسجيل الدخول
7. WHEN يضغط الأدمن على "رفع الحظر"، THE System SHALL إعادة تفعيل الحساب وتمكينه من تسجيل الدخول
8. IF حاول مستخدم غير أدمن الوصول إلى لوحة التحكم، THEN THE System SHALL إعادة توجيهه إلى الصفحة الرئيسية

---

### المتطلب 8: التحقق من النطاقات الجامعية

**قصة المستخدم:** بصفتي نظاماً، أريد التحقق من شرعية الإيميلات الجامعية، حتى تبقى المنصة حكراً على الطلاب الجامعيين.

#### معايير القبول

1. THE System SHALL الاعتراف بنطاقات الإيميل التالية كنطاقات جامعية شرعية: جميع النطاقات التي تنتهي بـ `.edu`، و`.ac.uk`، و`.ac.jp`، و`.edu.sa`، و`.edu.eg`، و`.edu.jo`، و`.edu.kw`، و`.edu.ae` وما شابهها من النطاقات الأكاديمية الدولية
2. WHEN يحاول مستخدم التسجيل بإيميل gmail أو yahoo أو hotmail أو ما شابه، THE System SHALL رفض التسجيل
3. THE System SHALL إجراء التحقق من النطاق في الخادم (Supabase Edge Function) وليس فقط في الواجهة الأمامية

---

### المتطلب 9: تجربة المستخدم والاستجابة

**قصة المستخدم:** بصفتي مستخدماً، أريد واجهة سريعة وسهلة الاستخدام على مختلف الأجهزة، حتى أتمكن من تصفح الموقع بشكل مريح.

#### معايير القبول

1. THE System SHALL عرض الموقع بتصميم متجاوب يعمل على الشاشات الكبيرة والمتوسطة والصغيرة (موبايل)
2. WHEN يتم تحميل الصفحة الرئيسية، THE System SHALL عرض حالة تحميل (loading state) أثناء جلب المشاريع
3. WHEN تفشل عملية جلب البيانات، THE System SHALL عرض رسالة خطأ واضحة للمستخدم
4. IF كانت قائمة المشاريع فارغة، THEN THE System SHALL عرض رسالة توضح أنه لا توجد مشاريع بعد

---

### المتطلب 10: دعم اللغات المتعددة

**قصة المستخدم:** بصفتي مستخدماً، أريد تصفح الموقع بلغتي المفضلة، حتى أفهم المحتوى بشكل أفضل.

#### معايير القبول

1. THE System SHALL دعم ثلاث لغات: العربية والإنجليزية والتركية
2. THE System SHALL عرض أزرار تبديل اللغة في شريط التنقل لجميع الصفحات
3. WHEN يختار المستخدم لغة، THE System SHALL تطبيق اللغة على جميع نصوص الواجهة فورياً وحفظ الاختيار
4. WHEN تكون اللغة عربية، THE System SHALL تطبيق اتجاه النص من اليمين لليسار (RTL) تلقائياً
5. WHEN تكون اللغة إنجليزية أو تركية، THE System SHALL تطبيق اتجاه النص من اليسار لليمين (LTR)
6. THE System SHALL إدارة جميع النصوص عبر ملفات ترجمة مركزية (locales/) بدون نصوص مضمّنة في الكود

---

### المتطلب 11: دعم PWA (تطبيق قابل للتثبيت)

**قصة المستخدم:** بصفتي مستخدماً، أريد تثبيت الموقع كتطبيق على جهازي، حتى أتمكن من الوصول إليه بسرعة كأي تطبيق عادي.

#### معايير القبول

1. THE System SHALL توفير ملف `manifest.json` صحيح يتيح تثبيت الموقع كتطبيق
2. WHEN يزور المستخدم الموقع على متصفح يدعم PWA، THE System SHALL عرض خيار "تثبيت التطبيق"
3. THE System SHALL توفير Service Worker يخزن الأصول الثابتة للعمل بشكل جزئي دون اتصال
4. THE System SHALL توفير أيقونات التطبيق بأحجام مناسبة (192x192 و512x512)

---

### المتطلب 12: معايير جودة الكود

**قصة المستخدم:** بصفتي مطوراً، أريد كوداً نظيفاً وغير متكرر، حتى يكون الصيانة سهلة والكود بشرياً.

#### معايير القبول

1. THE System SHALL يكون الكود المنشور خالياً تماماً من التعليقات
2. THE System SHALL عدم تكرار أي منطق — كل دالة أو وحدة تُكتب مرة واحدة فقط وتُستدعى من أماكن متعددة
3. THE System SHALL استيراد الشعار دائماً من `assets/logo.svg` في جميع الصفحات
